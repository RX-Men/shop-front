# Sprint 3: Cart Page & Commercetools Integration - 22.06.2026

## Что было сделано

Основной задачей спринта стала разработка страницы корзины (Cart Page) и первая полноценная интеграция приложения с
Commercetools.

Работу разбил на несколько этапов и постепенно двигался от локальной реализации корзины к работе с реальным API.

За спринт удалось:

- создать страницу корзины и подготовить её структуру;
- реализовать CartService на Angular Signals;
- связать карточки товаров с корзиной через события добавления товара;
- добавить поддержку SKU в модель товара для интеграции с Commercetools;
- подключить Commercetools SDK и настроить клиент для работы с API;
- реализовать создание корзины через Commercetools;
- реализовать добавление товаров в корзину через SKU;
- начать интеграцию отображения реальных данных корзины на странице;
- подготовить обработчики изменения количества товаров и удаления товаров из корзины.

Отдельно пришлось довольно много работать с документацией Commercetools и разбираться в структуре сущностей платформы:
Cart, LineItem, SKU, Update Actions и механизме версионирования корзины.

## Проблемы

Основные сложности этого спринта были связаны не столько с Angular, сколько с Commercetools.

Сначала возникли проблемы с авторизацией и правами доступа. Было неочевидно, какой именно тип токена требуется для
работы с корзиной и какие scope необходимо добавить клиенту.

После настройки anonymous session flow появились ошибки с добавлением товаров в корзину. Выяснилось, что для работы API
необходим SKU существующего опубликованного продукта. После исправления SKU появилась новая проблема — отсутствие Tax
Category у товаров, из-за чего Commercetools отказывался создавать Line Item.

Ещё одной сложностью стала работа с анонимной корзиной. После успешного создания корзины и добавления товара данные
отображались корректно, однако после обновления страницы корзина терялась на стороне клиента. Пришлось разбираться с
anonymous session, active cart и механизмом восстановления корзины между сессиями.

Также столкнулся с ограничениями прав доступа при попытке изменять количество товаров и удалять товары из корзины.
Некоторые запросы успешно выполнялись, а некоторые возвращали ошибки недостаточных прав, несмотря на корректную
реализацию на стороне фронтенда.

## Решения

- Для управления состоянием корзины использованы Angular Signals.
- Добавление товаров реализовано через SKU и реальные запросы к Commercetools API.
- Для хранения идентификатора корзины использован localStorage.
- Настроен anonymous session flow для работы без авторизации пользователя.
- Исправлены ошибки, связанные с отсутствием SKU у товаров.
- Исправлены ошибки, связанные с отсутствием Tax Category в продуктах Commercetools.
- Для диагностики запросов активно использовались DevTools и Network вкладка браузера.
- Изучена документация Commercetools по работе с Cart API и My Carts API.

## Что я узнал

Впервые поработал с полноценной e-commerce платформой и получил представление о том, как устроена работа корзины на
стороне реального интернет-магазина.

Разобрался с назначением SKU и тем, почему именно этот идентификатор используется для добавления товаров в корзину.

Лучше понял концепцию optimistic locking и использование version при обновлении сущностей в Commercetools.

Получил практический опыт работы с внешним SDK, чтением документации и поиском причин ошибок через ответы API.

Также стало понятнее, что далеко не все проблемы решаются кодом на фронтенде — часть функциональности зависит от
настроек проекта, ролей и прав доступа во внешней системе.
